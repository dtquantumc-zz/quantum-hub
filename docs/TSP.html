Traveling Saleperson Code¶

TravelerSalesPerson.TSP_solver(G)¶

The TSP_solver function actually uses Ocean SDK and a call to dwave’s computer to solve the Traveling Salesperson Problem.

Parameters

G – This is the networkx graph to be solved. Usually the output of make_graph is directly plugged into this.

Returns

It returns the route as a List, a directed networkx graph, and the cost of the route.

TravelerSalesPerson.make_graph(cities, list_id)¶

This function creates the TSP graph that will be solved. Distances are measured in miles for Cities.

Cities:

  1. Albuquerque

  2. Boston

  3. Charlotte, NC

  4. Detroit

  5. Evanston, IL

  6. Frankfort, Kentucky

  7. Gulfport, Mississippi

  8. Toronto, ON, CA

  9. Vancouver, BC, CA

  10. Winnipeg, MB, CA

  11. Calgary, AB, CA

  12. Regina, SK, CA

  13. Montréal, QC, CA

Vancouver locations:

  1. UBC campus

  2. SFU campus

  3. Canada Place

  4. Rogers Arena

  5. Stanley park

  6. Metropolise at Metrotown

  7. Lafarge lake

  8. Burnaby lake

  9. Queen Elizabeth park

Parameters

  • cities – This is a list of indices of cities to be included in the problem.

  • list_id – This indicates which city-set is being used. 0 for Cities, 1 for Vancouver locations.

Returns

Returns the graph as a networkx graph.
